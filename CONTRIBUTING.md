# Contributing to SIEM/SOAR Detection Engineering

Contributions, corrections, and additions are welcome — this is a living reference, not a finished product.

## How to Contribute

1. **Open an issue first** for anything beyond a small fix (typo, broken link) — this avoids duplicate effort on larger additions like new detection rules or playbooks.
2. **Fork the repository** and create a branch for your change.
3. **Keep changes scoped** — one topic per pull request makes review faster and keeps history readable.
4. **Follow existing structure and naming conventions** in the relevant folder (see the README for the folder map).
5. **Submit a pull request** with a clear description of what changed and why.

## What's Especially Welcome

- Real-world tuning notes on existing detection rules (false-positive sources you've encountered, threshold adjustments)
- Corrections to technical inaccuracies
- Additional platform translations for detection logic (e.g., a Splunk SPL equivalent for a Sigma rule that only has KQL)

## Code of Conduct

By participating, you agree to uphold the standards in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
